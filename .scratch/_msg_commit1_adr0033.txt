feat(filename): adopt trailing-slug grammar (ADR-0033 · fcop 1.6.0)

Formally consolidate the trailing-slug filename pattern that emerged
22+ times in the codeflow project (e.g. TASK-*-PM-to-OPS-phase-a.md)
into FCoP's filename grammar. MINOR additive extension, fully backward
compatible: every pre-1.6 filename continues to parse with slug=None.

Protocol layer:
- fcop_protocol_version: 2.3.0 -> 2.4.0
- fcop / fcop-mcp packages:   1.5.1 -> 1.6.0
- fcop_rules_version: unchanged (slug is commentary-layer additive)

Grammar:
  _SLUG = [a-z](?:[a-z0-9-]*[a-z0-9])?
  TASK-{date}-{seq}-{sender}-to-{recipient}[.{slot}][-{slug}].md
  REPORT-{date}-{seq}-{reporter}-to-{recipient}[-{slug}].md
  ISSUE-{date}-{seq}-{reporter}[-{slug}].md

Lowercase-start enforced for disambiguation with _ROLE (which allows
e.g. OPS-001); trailing hyphen forbidden for aesthetic. slug does NOT
participate in routing -- list_tasks(recipient="OPS") matches both
plain and slug-suffixed task filenames.

Code (src/fcop/core/filename.py):
- Add _SLUG regex + exported SLUG_RE
- Append (?:-(_SLUG))? to TASK/REPORT/ISSUE filename patterns
- Extend TaskFilename / ReportFilename / IssueFilename dataclasses
  with optional slug: str | None = None
- parse_*_filename returns slug; build_*_filename accepts slug kwarg
- _check_components validates slug against SLUG_RE

Tests (tests/test_fcop/test_core_filename.py):
- 31 new tests across TestTaskTrailingSlug / TestBuildTaskFilenameWithSlug
  / TestReportTrailingSlug / TestIssueTrailingSlug / TestSlugRegexExport
- Backward compatibility: legacy filenames parse with slug=None
- Reject invalid slugs (uppercase start, digit start, underscore, dot,
  trailing hyphen, space, etc.)
- _ROLE vs slug disambiguation pinned
- Full suite: 1057/1057 passing (zero regression)
