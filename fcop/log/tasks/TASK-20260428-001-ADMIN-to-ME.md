v3 已补上附录 A（适用边界）与 "你读完后可以做的 5 件事"，文章 703 行。

但 ADMIN 亲自读过之后指出两点问题：

1. **头部太重**——v2/v3 把「三段公式」「读者画像三框」「为什么你需要这份教程」都塞进头部。对一个还不知道 fcop-mcp 是什么的读者来说，是「先喉心鸡汤后看 demo」，门槛高。违反「先过程 · 后彩蛋」的教学论。

## 重头戏：过程里「**大量贴 agent 自留笔记**」

ADMIN 明说：“**多讲一些 agent 自己做的笔记；我们只要自然语言，他把 admin 的需求也记录了**”。

这意味着每个过程章节里都要出现这个对比：

```
【ADMIN 只说了这一句自然语言】
"帮我写个贪吃蛇。”

【agent 自动落盘了这些文件（真实片段）】
TASK-20260427-001-PLANNER-to-CODER.md
──────────────────────
subject: 实现贪吃蛇核心逻辑
body: ...
```

贴哪些？

- 章节 2（单人模式）：贴 agent solo 初始化后自动生成的 TASK-001 / REPORT-001 的 frontmatter + 正文片段。
- 章节 3（多人模式）：贴 PLANNER 越界事故里 agent **自己写的** ISSUE-20260427-001-PLANNER.md 的完整忏悔片段（这是最生动的 "agent 笔记证据」）。
- 也可以贴一段 docs/agents/log/ 里归档后的 TASK 文件名列表作为「你看，agent 狗饣得有多多」。
