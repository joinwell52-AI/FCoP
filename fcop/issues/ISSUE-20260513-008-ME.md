# ISSUE-008: `fcop_audit` 1.5.1 桶豁免逻辑系统性缺失

## 严重性

**P1 medium**（不阻塞 audit 主功能 / 但导致 protocol-compliant 项目出现持续性 false-positive P1-002 violation / 需 ADMIN 决策接受 lingering / 影响「Mode 1 完全合规」类 sprint DoD）

## 问题摘要

`fcop_audit(scope='takeover')` 在 1.5.1 中对 **3 类历史桶** 内的 protocol-compliant 文件不豁免，仍报 `P1-002 (Rule 2 桶错位)` violation。CodeFlow 项目 Mode 1 sprint 中 4 个不同 sub-case 实证，建议升级 audit 为「按 kind/桶/命名三对齐 + 桶豁免清单」逻辑。

---

## Sub-case 1: `_archive/` 桶内合规文件仍报 P1-002

**现象**: `fcop/tasks/_archive/<month>/TASK-*.md` 文件 frontmatter `kind: task` + 桶路径 `tasks/_archive/<month>/` + 命名 `TASK-` 三对齐 → **audit 仍报 P1-002**

**真因**: audit 1.5.1 对 `_archive/` 子桶不豁免 / 应识别为「历史 archived task / 合规」

**CodeFlow 实证**:
- Mode 1 Phase A 22:51 OPS-025 第 2 轮 GATE 8 FAIL P1-002=1 件
- 文件: `fcop/tasks/_archive/2026-04/TASK-20260425-002-ADMIN-to-PM.md`
- ADMIN D48=甲 接受为 known upstream lingering

---

## Sub-case 2: `log/legacy-non-protocol/` 桶内文件仍报 P1-002

**现象**: archive 任意 `kind: task` 或 `kind: report` 文件到 `fcop/log/legacy-non-protocol/` → **audit 仍报 P1-002**

**真因**: audit 1.5.1 对 `log/legacy-non-protocol/` 桶不豁免 / 应识别为「legacy historical / 合规」

**CodeFlow 实证**:
- Mode 1 Phase B 23:09 OPS-026 archive 后 P1-002=3 件（新增 2 件）
- 文件: `fcop/log/legacy-non-protocol/TASK-20260511-027-PM-to-OPS.md` + `REPORT-20260511-027-OPS-to-PM.md`
- ADMIN D50=甲 接受为 known upstream lingering

---

## Sub-case 3: PM→ADMIN flow `mv` 命令推荐错位

**现象**: 历史 `TASK-*-PM-to-ADMIN.md` 文件 frontmatter `kind: task` → audit 推荐 `mv tasks/ → reports/`（方向错）

**真因**: PM→ADMIN flow 协议演化后**应当**是 `kind: report`（不是 `kind: task`），audit 命中桶错位但 remediation 命令方向错（应 patch frontmatter `kind` + rename 文件 / 不是 mv）

**CodeFlow 实证**:
- Mode 1 Phase A 22:35 OPS-024 第 1 轮 GATE 8 FAIL P1-002=4 件
- 项目侧 manual fix：StrReplace frontmatter + rename（D47=乙）→ `9d8c4bc` + `300261d`

---

## Sub-case 4: audit summary `p1_count` 含义歧义（group vs file）

**现象**: audit summary `p1_count=2` 实际代表 2 个 **violation group**（P1-001 + P1-002 各 1 group）/ 实际 lingering 文件数为 5 件 → 多次混淆

**真因**: audit summary 字段仅含 group_count / 不含 file_count / 阅读者默认为 file_count

**CodeFlow 实证**: QA-027 INSPECTION-003 §5.1 揭示（QA I-6 候选）

---

## 建议 patch（按优先序）

1. **新增 `audit-ignore` 机制**: 项目侧可在 `fcop/.audit-ignore.yaml` 声明 known lingering / audit 分离输出 `accepted_lingering` vs `pending_remediation`
2. **桶豁免清单**: 默认豁免 `_archive/` + `log/legacy-non-protocol/` + `log/archive/` / 对桶内 protocol-compliant 文件不报 P1-002
3. **summary 字段升级**: 同时输出 `violation_group_count` + `violation_file_count`
4. **PM→ADMIN flow grace**: 对历史 `TASK-*-PM-to-ADMIN.md` + `kind: task` 给 warning（建议 rename+patch）/ 不报 P1-002
