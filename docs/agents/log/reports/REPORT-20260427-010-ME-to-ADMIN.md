### 每章末尾小结（6 处）

- 第 0 章：指挥 agent 的第一步是装工具 / 认版本是 ADMIN 基本功 / MCP 接上≠项目就绪
- 第 1 章：四步循环不能跳 / 文件名带身份是审计不是装饰 / 角色占用从落盘扫出不是自我声明
- 第 2 章：重组团队不应删历史 / 代码-产品-历史不被团队结构变动 / UNBOUND 默认是对你有利的
- 第 3 章：ADMIN 提需不必说全 / 派单=write_task / 写完 TASK≠“可以接着干”（伏笔）
- 第 4 章（重点）：LLM 真实失败模式 #1 是 solo 惯性 / 边界检查需三层兑底 / 越界发生要归档不要删除
- 第 5 章：接手不是橡皮图章 / 接管姿态要言语明确 / 同 workspace 可多角色升级不丢记忆

### 总章「指挥 agent 的 6 条铁律」（一记安被为心）

1. 先识别再分配
2. 永远走四步循环
3. 一个 session 只演一个角色
4. 不删过往只追加
5. 落盘是写给未来的自己看的
6. 越界发生时立刻归档成 ISSUE

+ 隐形第 7 条：**识别记忆爆炸的 5 个征兆**（重复问/忘事/自相矛盾/你想“我自己来”/上下文加载变慢）

每条铁律后跟 "在 fcop-mcp 里" + "换别的工具怎么对应" 两句括号，保证不依赖 FCoP 也能带走。
