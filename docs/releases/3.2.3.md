# fcop 3.2.3 — 团队模板与文档同步 · FCoP 3.0 合规

**发布日期**：2026-05-23  
**包版本**：`fcop==3.2.3` · `fcop-mcp==3.2.3`（双包 lockstep，per [ADR-0002](../../adr/ADR-0002-package-split-and-migration.md)）  
**PyPI**：<https://pypi.org/project/fcop/3.2.3/> · <https://pypi.org/project/fcop-mcp/3.2.3/>

---

## 一句话

**把 bundled 团队模板和规则文档里残留的 v2 目录引用（`tasks/` / `reports/` / `log/`）全面迁到 FCoP 3.0 的 `_lifecycle/` 拓扑，并新增 `fcop` 库专用发布前检查脚本。**

无 API 破坏性变更；已有 v3 项目升级后应跑 `redeploy_rules()` 刷新本地规则四件套。

---

## 为什么有这个版本

3.2.0–3.2.2 在**代码层**完成了 v3 生命周期工具、深度历史归档和 MCP 发布前检查硬化，但 **bundled 团队样板**（`letter-to-admin`、`roles/*`、`TEAM-OPERATING-RULES` 等）和项目根 **`.cursor/rules/`** 仍引用 v2 五桶路径。新用户按模板初始化或阅读 ADMIN 说明书时，会看到与 FCoP 3.0 规范不一致的目录说明。

3.2.3 是一次 **文档 + 模板合规 patch**，把「代码已是 v3、文档还在说 v2」的漂移清掉。

---

## 变更摘要

### Fixed — bundled 团队模板（`fcop`）

| 文件 / 范围 | 变更 |
|---|---|
| `letter-to-admin.{zh,en}.md` | MCP 工具数 **32 → 45**；目录引用改为 `_lifecycle/` |
| 四套团队 `roles/*.md`（`dev-team` / `media-team` / `mvp-team` / `qa-team` / `solo`） | `fcop/tasks/` → `_lifecycle/inbox/`；`log/` → `_lifecycle/archive/` |
| `TEAM-OPERATING-RULES.{md,en.md}` | v2 目录模型 → v3 生命周期阶段模型 |
| 团队 `README.{md,en.md}` | 目录引用对齐 `_lifecycle/` |
| `.cursor/rules/fcop-rules.mdc` / `fcop-protocol.mdc` | 与 bundled 版本同步（此前本地副本漂在 v3.0.0） |

### Added — `fcop`

- **`scripts/fcop_prerelease_check.py`** — `fcop` 库专用发布前检查（10 项）：
  - 版本号一致性（`_version.py`）
  - 关键 bundled 文件存在性
  - 规则文件 frontmatter / `_lifecycle/` 文档覆盖
  - 团队模板存在性与 v2 遗留路径扫描
  - `.cursor/rules/` 与 bundled 规则文件大小近似一致

> 注：`fcop-mcp` 侧的 `mcp/prerelease_check.py` 在 **3.2.2** 已扩展 Check 7–10；3.2.3 补的是 **`fcop` 库本体**的平行检查脚本。

## 功能 / API 变更

**无运行时 API 变更。** 行为变化仅体现在：

- 新初始化项目读到的团队模板路径描述正确
- `letter-to-admin` 工具数与实际 MCP 工具表一致（45）
- 发版维护者可用 `python scripts/fcop_prerelease_check.py` 做发版前自检

## 升级指引

### 从 3.2.x 升级

```bash
pip install -U "fcop>=3.2.3,<3.3" "fcop-mcp>=3.2.3,<3.3"
```

或在项目目录（ADMIN 会话）：

```
redeploy_rules()
```

刷新 `.cursor/rules/fcop-rules.mdc`、`.cursor/rules/fcop-protocol.mdc`、`AGENTS.md`、`CLAUDE.md`。

### 从 2.x / 早期 3.0.x 升级

仍须先完成 v3 拓扑迁移（若尚未做）：

```bash
python -m fcop migrate --to-v3
fcop_audit(scope="upgrade")
```

详见 [`docs/MIGRATION-3.0.md`](../MIGRATION-3.0.md)。

1. 升级到包含修复的后续 patch（若已发布），或  
2. 从本仓库 `src/fcop/rules/_data/fcop-protocol.mdc` 复制，或  
3. 在项目里执行 `redeploy_rules()`（绑定已修复的 `fcop` 包版本后）

## English summary

**fcop 3.2.3** is a **documentation and bundled-template compliance patch** for FCoP 3.0:

- All preset team templates now reference `_lifecycle/{inbox,active,review,done,archive}/` instead of legacy `tasks/` / `log/` paths.
- `letter-to-admin` tool count corrected to **45**.
- Project-local `.cursor/rules/*.mdc` synced with bundled rules.
- New `scripts/fcop_prerelease_check.py` (10 checks) for the `fcop` library release gate.

No breaking API changes. After upgrade, run `redeploy_rules()` to refresh host-neutral rule files.

Full changelog: [`CHANGELOG.md`](../../CHANGELOG.md#323--2026-05-23-team-template--documentation-sync--fcop-30-compliance).
