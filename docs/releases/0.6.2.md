# Release 0.6.2 — documentation & install alignment (lockstep)

- **Date**: 2026-04-25
- **PyPI (after upload)**: [`fcop 0.6.2`](https://pypi.org/project/fcop/0.6.2/) · [`fcop-mcp 0.6.2`](https://pypi.org/project/fcop-mcp/0.6.2/)

## What shipped

- **`fcop` 0.6.2** — no public library API or compat-CLI change vs 0.6.1; `pyproject.toml` **summary + Documentation** URL updated; PyPI **long** description from [`fcop-README.pypi.md`](../fcop-README.pypi.md) (library-only, not the monorepo root `README.md`). **Lockstep** with `fcop-mcp` per [ADR-0002](../adr/ADR-0002-package-split-and-migration.md).
- **`fcop-mcp` 0.6.2** — **packaging + docs** on the PyPI project page: `description` / `Documentation` link; unified customer install in [`mcp/README.md`](../../mcp/README.md) (recommended venv + `python -m fcop_mcp`, `uvx fcop-mcp` caveats, Windows / macOS `mcp.json`, verify commands, wrong-`fcop` warning). **MCP tool/resource contract** unchanged (ADR-0003).

## Verify after upload

```bash
python -m venv .venv
. .venv/Scripts/activate   # or source .venv/bin/activate
pip install -U "fcop==0.6.2" "fcop-mcp==0.6.2"
python -c "from fcop import Issue, Project; print('fcop', Issue.__name__)"
python -c "from fcop_mcp.server import mcp; print('ok')"
```

## Release engineering (maintainers)

- 首次为 `v0.6.2` 打 tag 时，`.github/workflows/release.yml` 里 **verify** 曾用「文件里第一个双引号串」对版本号；`src/fcop/_version.py` 文档行里的 **`"semver 承诺"`** 会抢先被匹配，导致 **verify 失败**（与 PyPI 无关）。已在后续提交中改为**只解析 `^__version__ = ...` 行**。维护者见 [release-process.md](../release-process.md) 中的说明。

## Background

PyPI does not allow replacing an existing file version; **refreshing** install docs for `pip` / project pages requires a **new patch release**. This release exists so end users can follow one canonical path and self-check a correct `fcop` install.
