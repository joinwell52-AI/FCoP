# fcop 1.0.1 / fcop-mcp 1.0.1 — Release Notes

**Release date:** 2026-05-10  
**Type:** Patch — documentation, bundled assets, and MCP resource additions. Zero behavioral change to existing tools or protocol rules.

---

## Why this release

`v1.0.0` was tagged on 2026-05-09 as the GA release of the FCoP v1.0 protocol.
Eleven subsequent commits on `main` updated bundled rule files, protocol spec documents,
and documentation to complete the **AI OS positioning** narrative and the
**seven core concepts** stabilisation that are the centrepiece of v1.0.
This patch bundles all of those changes and ships two new MCP read-only resources
so agents can look up the full protocol spec on demand without leaving the IDE.

---

## What changed

### New — `fcop://spec` and `fcop://spec/en` MCP resources

Two new read-only MCP resources are now available:

| Resource URI | Content |
|---|---|
| `fcop://spec` | FCoP v1.0 full spec — Simplified Chinese (informative reference translation) |
| `fcop://spec/en` | FCoP v1.0 full spec — English (authoritative; takes precedence when the two conflict) |

Both spec files are bundled inside the wheel (`fcop/rules/_data/fcop-spec-v1.0.{zh,en}.md`)
so they work offline and are version-locked to the installed package.

New public API: `fcop.rules.get_spec(lang: Literal["zh", "en"]) -> str`

### Updated — Bundled rule files (rules v2.1.0 / protocol v1.9.0)

`fcop-rules.mdc` (v2.1.0) now includes:

- **AI OS protocol stack diagram** — FCoP's position between Host/LLM inference and application code
- **Seven core concepts table** — Agent · IPC · Encoding · Event · Failure · Boundary · Audit with Chinese + English definitions

`fcop-protocol.mdc` (v1.9.0) now includes:

- **Rule 9 complete commentary** (§9.1–9.4): REVIEW file naming, Boundary layers, Failure 4-type × 5-action table, Event 12-type reference

### Updated — Workspace path (`docs/agents/` → `fcop/`)

Global path migration completed across 49 files:
letter templates, install prompts, all 41 team role templates (solo / dev / media / mvp / qa, both languages), essays, `docs/mcp-tools.md`, `fcop-README.pypi.md`.

Historical files (CHANGELOG history, old release notes, ADRs, runtime agent logs) intentionally left unchanged — their historical context makes the old path accurate.

### Updated — Protocol spec documents

- New: `spec/fcop-runtime-protocol-v1.0.zh.md` — Simplified Chinese parallel version of the normative English spec (informative)
- `spec/fcop-spec.md` updated to point to the v1.0 spec files as the current authoritative entry point

### Updated — PyPI metadata

- Both `fcop` and `fcop-mcp`: `Development Status :: 4 - Beta` → `5 - Production/Stable`
- `fcop-mcp` description updated: "depends on fcop 0.7.x" → "fcop 1.x"
- `mcp/README.md` Stability section updated to 1.x; upgrade notes point to `docs/MIGRATION-1.0.md`

### Fixed — Broken internal links (8 categories)

- `fcop-protocol.mdc`: `docs/essays/fcop-position.md` → `spec/fcop-runtime-protocol-v1.0.md`
- `mcp/README.md`, `docs/tutorials/README.md`: `docs/fcop-standalone*.md` → `docs/getting-started*.md`
- `essays/`: `../primer/fcop-primer.md` → `docs/getting-started.md`
- `README.md` / `README.zh.md`: directory tree and `fcop-standalone` links updated

## Upgrade

```bash
pip install -U fcop fcop-mcp
```

Then in a new session call `redeploy_rules()` to refresh local rule files.

For projects upgrading from `0.7.x`, see [`docs/MIGRATION-1.0.md`](../MIGRATION-1.0.md).

---

## Compatibility

- No breaking changes to any MCP tool signature or protocol rule behaviour
- `fcop://spec` and `fcop://spec/en` are purely additive
- All existing `fcop://rules`, `fcop://protocol`, and other resources unchanged
