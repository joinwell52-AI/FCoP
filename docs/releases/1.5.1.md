# fcop 1.5.1 — 版本字符串对齐补丁

**发布日期**：2026-04-25
**包版本**：`fcop==1.5.1`（`fcop-mcp` 不动，仍为 1.5.0）

---

## 为什么有这个 patch

`v1.5.0` 发布后，在进行版本对齐审计时发现：

1. `letter-to-admin.zh.md` 和 `.en.md` 的"规则版本"行写的是
   `fcop-protocol.mdc 2.2.0`，实际 v1.5.0 已将协议文档升至 `2.3.0`。
2. `tests/test_fcop_mcp/test_server.py` 里 `init_project` 测试仍检查
   旧版本字符串 `"v1.3.0 摘要"`，而 letter 已升至 `v1.5.0 摘要`，
   导致测试失败（1 failed in 1008）。

这两处均属发版流程 **Phase 3-V（版本字符串全文对齐）** 遗漏。

## 根因与改进

根本原因：`v1.5.0` 发版时 Phase 3-V 执行不完整，`fcop-protocol.mdc`
版本号从 `2.2.0` 升到 `2.3.0` 后，`letter-to-admin` 里的引用未同步更新。

改进措施：`RELEASE-CHECKLIST.md` 已加强 Phase 3-V 检查列表（于 v1.5.0 周期
随文档同步整改方案 `protocol-docs-sync-20260512.md` 引入），要求发版前逐行
人工检查版本字符串。本次 patch 是该改进措施落地后的第一次应用验证。
