# fcop 3.2.4 — PyPI 元数据与 bundled 协议编码修复

**发布日期**：2026-05-27  
**包版本**：`fcop==3.2.4` · `fcop-mcp==3.2.4`（双包 lockstep，per [ADR-0002](../../adr/ADR-0002-package-split-and-migration.md)）  
**PyPI**：<https://pypi.org/project/fcop/3.2.4/> · <https://pypi.org/project/fcop-mcp/3.2.4/>

---

## 一句话

**修复 PyPI 3.2.3 上无法重传的两类问题：wheel 内 `fcop-protocol.mdc` UTF-8 损坏，以及 `fcop-mcp` 项目链接「FCoP (协议)」指向已删除文档导致 404。**

无 API / 协议 schema 变更；请从 **3.2.3 升级到 3.2.4**（不要指望重装 3.2.3）。

---

## 背景

| 问题 | 现象 | 根因 |
|------|------|------|
| Bundled 协议乱码 | 安装 3.2.3 后 `.cursor/rules/fcop-protocol.mdc` 中文变 `鍗忚…` | v3.2.3 发布同步时编码损坏打进 wheel |
| PyPI 链接 404 | 侧边栏「FCoP (协议)」打不开 | `mcp/pyproject.toml` 仍指向 v1.0.0 已删的 `fcop-standalone.en.md` |
| PyPI 长描述过时 | 页面仍写 v2.0.2 / 35 工具 | `mcp/README.md` 未随 3.2.3 发版同步到 PyPI |

PyPI **禁止同版本覆盖**，只能发 **3.2.4** patch。

---

## 变更摘要

### Fixed — `fcop`

- 从 v3.2.2 干净副本恢复 `src/fcop/rules/_data/fcop-protocol.mdc`，保留 v3.2.3 协议内容与版本记录。
- 仓库内 `.cursor/rules/fcop-protocol.mdc` 与 bundled 源字节一致。

### Fixed — `fcop-mcp`

- `"FCoP (protocol)"` URL → `docs/getting-started.en.md`
- `mcp/README.md` → v3.2.3 文案（45 工具、`_lifecycle/`、lockstep 安装）

---

## 发版门禁

- `scripts/fcop_prerelease_check.py` **[11]**：检测 bundled / wheel 内 `fcop-protocol.mdc` 的 UTF-8 与已知乱码片段
- `scripts/release_audit.py` **R13** + CI `release.yml` smoke：安装后须含「协议解释」、不得含 3.2.3 事故乱码

---

## 升级指引

```bash
pip install -U "fcop>=3.2.4,<3.3" "fcop-mcp>=3.2.4,<3.3"
```

在 IDE 中执行 MCP 工具 **`redeploy_rules()`**（或 Python：`Project.deploy_protocol_rules(force=True)`），刷新项目根四份规则文件。

若曾安装 **3.2.3** 且本地协议文件已乱码，`redeploy_rules()` 会用 wheel 内正确副本覆盖。

## English summary

**fcop 3.2.4** is a **packaging and PyPI-metadata patch**:

- Restores valid UTF-8 in the wheel-bundled `fcop-protocol.mdc` (3.2.3 PyPI wheels are bad; upgrade required).
- Fixes `fcop-mcp` project URL `"FCoP (protocol)"` → `docs/getting-started.en.md`.
- Refreshes PyPI long description (`mcp/README.md`) for 45 tools and v3 `_lifecycle/` layout.

No API changes. Prefer **yanking PyPI 3.2.3** if you have maintainer access.
