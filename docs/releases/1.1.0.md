## What 1.1.0 ships (5 fields)

### Field 1 — `Agent.layer` (ADR-0010)

`fcop.json` 的 `roles[i]` 加可选字段 `layer: "worker" | "governance" | "admin"`。默认 `worker`。

新校验：governance fission 拒绝、worker 不能审 governance、admin programmatic create 拒绝。

### Field 2 — `Task.risk_level` (ADR-0011)

`TaskFrontmatter` 加可选字段 `risk_level: "low" | "medium" | "high" | "irreversible"`。默认 `medium`。

`irreversible` 必须配 `requires_rollback_plan` 占位字段（1.1 仅检查存在，1.2 enforce）。

### Field 3 — `Review.decision = needs_human` (ADR-0012)

ADR-0009 已经在 1.0 引入 `Review.decision` 4 值 enum；1.1 加第五值 `needs_human`。

`decision: needs_human` review 必须配 `human_approval` 占位（ADR-0013）。

### Field 4 — `Review.human_approval` (ADR-0013)

Review frontmatter 加可选子对象，含 `approver` / `decision` / `approved_at` / `channel` / `comment` / `evidence`。

`approver` 必须是 `layer: admin` 角色（联动 ADR-0010）；`channel: 'manual_file_edit'` 必须有 `evidence.device_id` 或 `channel_attestation`。

新增 `Project.mark_human_approved()` API + `fcop.approve_review` MCP tool。

### Field 5 — `Skill.tools[]` risk metadata (ADR-0014)

`spec/schemas/skill.schema.json` 从 1.0 占位升到 1.1 完整字段。

`fcop` 库新增 Skill loader（`load_skills_dir()` / `validate_skill()`）+ `_data/skills/` 模板目录。

`fcop` 库**不**enforce 运行时拦截行为——这是 host runtime 的事；FCoP 仅提供 schema + loader，让任何 host 可消费。

## What stays the same

- 所有 1.0.0 行为 100% 保留
- 所有字段都有默认值——旧文件读取无感
- ADR-0003 公开面承诺：可选参数加在末尾、有默认值、不破坏
- 不 bump `core/schema.PROTOCOL_VERSION` 整数
- 不 bump `fcop-rules.mdc` / `fcop-protocol.mdc` 的 rules version
