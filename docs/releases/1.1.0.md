# 1.1.0 — Layer 3 Governance Policy & Skill Encoding (DRAFT outline · target ~D+90)

> **Status**: DRAFT outline — produced at ADR-0015 charter time. Once
> `fcop@1.1.0` is on PyPI the DRAFT marker is removed.
>
> **Scope**: 补完 Issue #2 中推迟到 v1.1+ 的字段，并启动 Layer 3
> Governance Policy 抽离的第一刀。所有 v1.1 改动严格 additive。

## Provenance

`fcop@1.1.0` 是 [ADR-0015 charter](../../adr/ADR-0015-fcop-1.0-ai-os-protocol-charter.md) §Non-Goals 推后到 v1.1 的项目兑现：

- **Issue #2 Field 2** (`Task.risk_level`) — Layer 3 Governance Policy 第一个落地点
- **Issue #2 Field 5** (`Skill.tools[]` risk metadata) — Encoding 抽象（[ADR-0021](../../adr/ADR-0021-encoding-abstraction.md)）的第一个 alternative encoding 模板
- **Layer 1 Core Principles 与 Layer 2 Runtime Protocol 切开**（rules.mdc 拆分）— charter §Consequences 留的债

每个改动配独立 ADR，所有 v1.0 行为 100% 保留。

## What 1.1.0 ships

### 1. `Task.risk_level` field（Issue #2 Field 2）

`TaskFrontmatter` 加可选字段 `risk_level: "low" | "medium" | "high" | "irreversible"`，默认 `medium`。

v1.1 仅作 **event tag**——通过 [ADR-0018 Event Model](../../adr/ADR-0018-event-model.md) 暴露 `TASK_RISK_TAGGED` 事件，由 host adapter 决定如何路由（不在协议层 enforce policy）。

`irreversible` 必须配 `requires_rollback_plan` 占位字段（仅检查存在；内容 enforce 推 v1.2）。

详见后续 ADR-0023 (待开)。

### 2. `Skill.tools[]` risk metadata（Issue #2 Field 5）

`spec/schemas/skill.schema.json` v1.1 落地完整 Skill schema。

`fcop` 库新增 Skill loader（`load_skills_dir()` / `validate_skill()`）+ `_data/skills/` 模板目录。

`fcop` 库**不**enforce 运行时拦截行为——这是 host runtime 的事；FCoP 仅提供 schema + loader。

详见后续 ADR-0024 (待开)。

### 3. Encoding alternative templates（[ADR-0021](../../adr/ADR-0021-encoding-abstraction.md) 兑现）

新增 `core/encoding/json_encoding.py` 作 alternative encoding **示例实现**（不是默认）：

- 用户可 `Project(encoding=JSONEncoding())` 切到 JSON
- 与 MarkdownEncoding 100% round-trip 互转
- 验证 ADR-0021 抽象层 contract 真正可用

详见后续 ADR-0025 (待开)。

### 4. Layer 1 / Layer 2 拆分（rules.mdc 第一次结构性拆分）

`fcop-rules.mdc` 拆为：

- `fcop-principles.mdc` — Layer 1 Core Principles（0.a / 0.b / 0.c）
- `fcop-rules.mdc` — Layer 2 Runtime Protocol（rule 1-7，原内容保留）

`fcop-protocol.mdc` 不变（仍是 Layer 4 Implementation 解释）。

详见后续 ADR-0026 (待开)。

## What stays the same

- 所有 1.0.0 行为 100% 保留
- 所有字段都有默认值——旧文件读取无感
- ADR-0003 公开面承诺：可选参数加在末尾、有默认值、不破坏
- 不 bump `core/schema.PROTOCOL_VERSION` 整数
- 7 抽象 contract 100% 保留（v1.1 仅在 Encoding / Audit 抽象上加内容）
