# 1.0.0 — AI OS Protocol Layer (DRAFT outline · target 2026-06-20 ~ 2026-07-04)

> **Status**: DRAFT outline — produced at ADR-0015 charter time, to be
> filled in as ADR-0016..0021 land. Once `fcop@1.0.0` is on PyPI the
> DRAFT marker is removed and this file becomes the canonical release
> record.
>
> **Scope**: 协议本体的 framing 升级 + 7 核心抽象的最小可运行内核。
> v1.0 的 framing 是 "FCoP = AI OS Protocol Layer"——POSIX 在 Unix 中
> 的位置；具体兑现是 7 抽象的最小语义冻结，不是 K8s 级完整 OS。

## The philosophy 1.0.0 ships with

> 「FCoP 是 agent 的协议，我们发现了他，而不是发明；而正好人类可以读懂。」

v1.0 第一次把这句话从 essay 抬升到协议级宪章——见 [ADR-0015 §FCoP is discovered, not invented](../../adr/ADR-0015-fcop-1.0-ai-os-protocol-charter.md#fcop-is-discovered-not-invented)。

三条对所有后续 v1.x 决策的 tiebreaker：

1. **FCoP belongs to agents, not to us.** 任何候选加进协议的东西必须先答：「agent 在共享文件系统里自由协作时，会自然做这件事吗？」答否就是发明，不属于 FCoP。
2. **Our job is observation + naming + formalization**, not design. 二选一时挑「有更多 field 证据的」，不挑「理论上更漂亮的」。
3. **Human readability is a bonus, not a goal.** Substrate 恰好是 filesystem + Markdown，人能读——这是 FCoP 最重要的副作用；但**为人类可读而设计**会让 FCoP 退化回文档格式。

最强的 field evidence 已经写进 [ADR-0021 §Open Knowledge Surface](../../adr/ADR-0021-encoding-abstraction.md)：真实 FCoP 项目里，agent 在没有任何 spec 指导的情况下自发涌现 5 类命名前缀（`GUIDE-` / `SPEC-` / `STATUS-` / `TEAM-` / `LETTER-`），全部遵守同一 grammar。这件事 v1.0 在协议层正式承认为 Encoding Abstraction 的 **Open Knowledge Surface**——与 IPC Surface 并列的"另一面"。

## What 1.0.0 ships

### 1. Framing 升级 — FCoP 是 AI OS 的 POSIX 层

| 旧 (0.7.x) | 新 (1.0.0) |
|---|---|
| File-based Coordination Protocol | **Agent Runtime Protocol — the AI OS protocol layer** |
| "AI 协作规则" | "AI OS 协议层 / Agent Runtime Protocol" |
| `fcop-mcp` 偶尔被叫 "kernel" | `fcop-mcp` = **Host Adapter**（与 libc 类比）|
| spec 含义模糊 | 5 层架构图 + 7 核心抽象表 + 5 术语永久冻结（见 [ADR-0015 §术语表](../../adr/ADR-0015-fcop-1.0-ai-os-protocol-charter.md)） |

### 2. 7 核心抽象的最小语义冻结

每个抽象由独立 ADR 落地：

| # | 抽象 | ADR | 关键产出 |
|---|---|---|---|
| 1 | **Agent**（Lifecycle + 身份） | [ADR-0020](../../adr/ADR-0020-agent-boundary-and-capability.md) | 5 状态枚举 + Boundary 占位 |
| 2 | **Encoding**（filename + frontmatter + 目录） | [ADR-0021](../../adr/ADR-0021-encoding-abstraction.md) | 抽象层 vs Reference Encoding 切开；Markdown 是 v1.0 唯一推荐实现 |
| 3 | **IPC**（Task / Report / Issue / Review envelope） | [ADR-0017](../../adr/ADR-0017-review-file-type-minimal.md) + 现有 | REVIEW 加入 4 类；envelope contract |
| 4 | **Event Model** | [ADR-0018](../../adr/ADR-0018-event-model.md) | 8 最小事件 + subscribe contract |
| 5 | **Failure Semantics** | [ADR-0019](../../adr/ADR-0019-failure-and-recovery-semantics.md) | 4 failure × 5 recovery + Session hook |
| 6 | **Boundary**（can / cannot capability） | [ADR-0020](../../adr/ADR-0020-agent-boundary-and-capability.md) | layer 升级为 capability bundle 简写 |
| 7 | **Audit**（REVIEW minimal surface） | [ADR-0017](../../adr/ADR-0017-review-file-type-minimal.md) | 4 decision enum；needs_human / human_approval **不在 v1.0**（推 v1.2） |

### 3. JSON Schema 形式化（7 抽象 × 7 schema）

`spec/schemas/` 落 7 份 JSON Schema 文件作为 single source of truth（[ADR-0016](../../adr/ADR-0016-json-schema-for-7-abstractions.md)）：

```
spec/schemas/
├── agent.schema.json
├── ipc-envelope.schema.json
├── encoding.schema.json
├── event.schema.json
├── failure.schema.json
├── boundary.schema.json
└── review.schema.json
```

`jsonschema>=4.0,<5.0` 进生产依赖。

### 4. 公开 API 1.0 lock + 版本号语义重定义

ADR-0003 留的 0.7→1.0 一次性清理窗口在此兑现。详细清理清单见 D+30 前补完。

| 维度 | 0.x 含义 | 1.x 含义 |
|---|---|---|
| MAJOR | N/A | 协议级 breaking + 至少 6 个月共存 + 官方迁移脚本 |
| MINOR | "新功能可能 break" | **只允许 additive**，新字段对旧 consumer 透明 |
| PATCH | "bug fix" | **绝对零行为变化**，仅 bug fix |

`fcop` 与 `fcop-mcp` 继续 lockstep MINOR。

### 6. Workspace 迁移：`docs/agents/` → `fcop/`（[ADR-0022](../../adr/ADR-0022-workspace-directory-convention.md)）

v1.0 把默认协议工作区从 `docs/agents/` 迁移到顶层 `fcop/`：

```
旧（0.7.x）：                       新（v1.0+）：
project/                           project/
├── src/                           ├── src/
└── docs/                          └── fcop/
    └── agents/                        ├── fcop.json
        ├── fcop.json                  ├── tasks/
        ├── tasks/                     ├── reports/
        └── ...                        ├── issues/
                                       ├── reviews/      ← REVIEW-* 新加
                                       ├── shared/
                                       └── log/
```

**迁移工具**：

```bash
fcop migrate-workspace               # dry run
fcop migrate-workspace --apply       # 执行（git mv 保留历史）
```

**兼容承诺**：
- 0.7.x 项目不会被破坏，只会得 warning
- `Project(workspace_dir="docs/agents/")` 显式调用永远合法（escape hatch 永久保留）
- 详见 [ADR-0022 §Backwards Compatibility](../../adr/ADR-0022-workspace-directory-convention.md)

- `fcop-rules.mdc` / `fcop-protocol.mdc` 正文不动（仅顶部 description 升级 framing）
