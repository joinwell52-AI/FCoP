# 1.0.0 — Protocol freeze (DRAFT outline · target 2026-05-23)

> **Status**: DRAFT outline — produced at ADR-0007 charter time, to be
> filled in as ADR-0008 + ADR-0009 land. Once `fcop@1.0.0` is on PyPI
> the DRAFT marker is removed and this file becomes the canonical
> release record.
>
> **Scope**: Pure structural release. No new business behaviour. The
> goal is to give FCoP a machine-readable spec, a stable public API,
> and a fourth file type (`REVIEW-*.md`) so the governance fields in
> 1.1.0 have somewhere to live.

## What 1.0.0 ships

### 1. JSON Schema as machine-readable spec (ADR-0008)

`spec/schemas/` 下落 5 份 `*.schema.json`：

```
spec/schemas/
├── agent.schema.json
├── task.schema.json
├── review.schema.json
├── session.schema.json    # 1.0 仅占位
└── skill.schema.json      # 1.0 仅占位
```

`jsonschema>=4.0,<5.0` 进生产依赖。

新增 `tests/test_schemas/`，每个 schema 至少 3 个 case；新增
`tests/test_fcop/test_schema_dataclass_alignment.py` 校验
`spec/schemas/*.schema.json` 与 `models.*` dataclass 字段集合一致。

### 2. REVIEW-* file type (ADR-0009)

文件名 grammar：

```
REVIEW-{YYYYMMDD}-{NNN}-{REVIEWER_ROLE}-on-{SUBJECT_REF_SHORT}.md
```

新增 `models.Review` dataclass + `Project.write_review() / read_review() / list_reviews() / archive_review()` + `core/frontmatter.parse_review_frontmatter()` + 默认目录 `docs/agents/reviews/` + 归档目录 `docs/agents/log/reviews/`。

### 3. Public API surface 1.0 lock (ADR-0007)

ADR-0003 留的 0.7→1.0 一次性清理窗口在此兑现：

- 任何在 0.7.x 标过 `DeprecationWarning` 的 API 真删
- 公开符号一致性 audit
- `models.*` dataclass 字段进入 1.x 只进不出锁

### 4. Version policy 1.x semantics (ADR-0007)

- MAJOR (1.x → 2.x)：协议级 breaking + 至少 6 个月共存 + 官方迁移脚本
- MINOR (1.0 → 1.1)：只允许 additive，新字段对旧 consumer 透明
- PATCH (1.0.0 → 1.0.1)：绝对零行为变化，仅 bug fix
- `fcop` 与 `fcop-mcp` 继续 lockstep MINOR

## What stays the same

- 所有 0.7.x 的 TASK / REPORT / ISSUE 文件 100% 兼容
- `fcop_report()` 输出格式不变
- `fcop-mcp` 22 个 MCP tool 接口不变
- `fcop-rules.mdc` / `fcop-protocol.mdc` 内容不动（rules version 不 bump）
- Cursor 用户 mcp.json 配置零修改
