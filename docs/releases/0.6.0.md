## 1. 一句话结论

0.6.0 是 FCoP 参考实现的**首个稳定版**，把 0.5.x 的单一包拆成 `fcop`（纯 Python 库）+ `fcop-mcp`（MCP 服务器）双包，并用五份 ADR 把协议级承诺全部敲死。自此进入 ADR-0003 定义的 additive-only 稳定期。

## 2. Shipping scope

### 2.1 包结构（ADR-0002 落地）

| 包 | 版本 | 用途 | Size | Runtime deps |
|---|---|---|---|---|
| `fcop` | 0.6.0 | 纯 Python 库，零 MCP 依赖 | 212 KB wheel | `pyyaml` |
| `fcop-mcp` | 0.6.0 | MCP 服务器，依赖 `fcop` | 22 KB wheel | `fcop>=0.6,<0.7`, `fastmcp>=3.2`, `websockets>=12` |

### 2.2 公共 API surface（ADR-0001 Accepted）

`fcop` 包的 4 个 surface 被 `tests/test_fcop/test_public_surface.py` 锁定：

- `fcop.__all__` — 顶层导出
- `Project` 类的 public methods / properties（含 `init` / `write_task` / `read_task` / `list_tasks` / `write_report` / `read_report` / `list_reports` / `write_issue` / `read_issue` / `list_issues` / `archive_task` / `inspect_task` / `deploy_role_templates` / `drop_suggestion` / `status`）
- `fcop.models` 的 dataclass 字段（`Task` / `Report` / `Issue` / `TeamConfig` / `ProjectStatus`）
- `fcop.teams` / `fcop.rules` 的导出

### 2.3 MCP tool / resource 合同（ADR-0003 Commitment #2 落地）

`fcop-mcp` 的 24 个 tool + 10 个 `fcop://` resource 被 `tests/test_fcop_mcp/test_tool_surface.py` 锁定，**与 0.5.4 完全兼容**。

24 tool 覆盖：init / workspace / task CRUD / report CRUD / issue CRUD / suggest / archive / inspect / deploy / teams / rules / check_update / upgrade_fcop / unbound_report。

## 4. 发布执行步骤（实际做的事）

### 4.1 本地发布路径（0.6.0 用的是这条）

```powershell
# 1. bump 版本
# src/fcop/_version.py、mcp/src/fcop_mcp/_version.py 都改成 "0.6.0"

# 2. CHANGELOG [Unreleased] → [0.6.0] - 2026-04-23

# 3. commit + build
git commit -am "chore(release): 0.6.0 ..."
Remove-Item -Recurse -Force dist, build, mcp/dist, mcp/build -ErrorAction SilentlyContinue
py -3.10 -m build
Push-Location mcp; py -3.10 -m build; Pop-Location

# 4. 验包
py -3.10 -m twine check dist/* mcp/dist/*

# 5. 干净 venv 冒烟
$v = "$env:TEMP\smoke-0.6.0"
py -3.10 -m venv $v
& "$v\Scripts\python.exe" -m pip install --upgrade pip --quiet
& "$v\Scripts\pip.exe" install (Get-ChildItem dist\fcop-*.whl).FullName
& "$v\Scripts\pip.exe" install (Get-ChildItem mcp\dist\fcop_mcp-*.whl).FullName
& "$v\Scripts\python.exe" -c "import fcop, fcop_mcp; print(fcop.__version__, fcop_mcp.__version__)"

# 6. 上传（先 fcop 后 fcop-mcp）
py -3.10 -m twine upload dist\fcop-0.6.0*
py -3.10 -m twine upload mcp\dist\fcop_mcp-0.6.0*

# 7. tag + push
git tag -a v0.6.0 -m "Release 0.6.0 ..."
git push origin feat/0.6-library-split
git push origin v0.6.0

# 8. GitHub Release
gh release create v0.6.0 --title "..." --notes "..."
```

### 4.2 自动化路径（0.6.1 起启用）

`.github/workflows/release.yml` 会在 `v*` tag push 后自动：

```
verify(tag↔version 一致性) → build(双包 wheel+sdist, twine check, smoke install)
  → publish-fcop(PYPI_TOKEN_FCOP) → publish-fcop-mcp(PYPI_TOKEN_FCOP_MCP)
  → github-release(抽 CHANGELOG 章节 + 4 份 artifact)
```

详见 `docs/release-process.md`。

## 5. 发布后观测

### 5.1 PyPI 验证

```bash
$ curl -s https://pypi.org/pypi/fcop/json | jq '.info.version'
"0.6.0"

$ curl -s https://pypi.org/pypi/fcop-mcp/json | jq '.info.version'
"0.6.0"
```

### 5.2 下游影响 —— **已知兼容断点**

| 场景 | 0.5.4 行为 | 0.6.0 后行为 | 评估 |
|---|---|---|---|
| `pip install fcop` | 装 MCP server + CLI `fcop` | 装纯库，**无 `fcop` 命令** | ⚠️ **破坏** |
| `uvx fcop` | 跑 MCP server | **command not found**（0.6.0 无入口） | ⚠️ **破坏** |
| MCP 配置 `command: uvx, args: [fcop]` | 能连上 | 失败 | ⚠️ **破坏** |
| `from fcop.server import ...`（私有 API） | 能 import | `ImportError`，`fcop.server` 已不存在 | 🟡 私有 API，允许破坏 |
| `docs/agents/tasks/*.md` 老文件 | — | 读得到（Grandfather 条款覆盖）| ✅ 兼容 |
| MCP tool 名 / 参数 / 返回格式 | 0.5.4 格式 | **完全一致**（ADR-0003 锁定） | ✅ 兼容 |

### 5.3 待修（进 0.6.1）

针对 5.2 三处 ⚠️，规划 **0.6.1 compat shim**：

- 在 `fcop`（库）包加一个 `fcop` console script，执行时**不**启动服务器，而是打印清晰的迁移指引：

  ```
  fcop 0.6+ is a pure Python library. The MCP server moved to a separate package.

      pip install fcop-mcp
      # then in Cursor / Claude Desktop mcp.json:
      #   "fcop": { "command": "uvx", "args": ["fcop-mcp"] }

  See the migration guide: https://github.com/joinwell52-AI/FCoP/blob/main/docs/MIGRATION-0.6.md
  ```

  并 `exit 1` 让老调用方的脚本/systemd 能捕获非 0 退出码。

- 这是 ADR-0003 允许的 additive 改动（`fcop` 多一个 script，不破坏库 API）。
- 实施计划：`src/fcop/_compat_cli.py` + `pyproject.toml` 加 `[project.scripts] fcop = "fcop._compat_cli:main"`，配 3 个单测。
