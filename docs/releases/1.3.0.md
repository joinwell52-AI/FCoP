**发布日期**：2026-05-12

## 为什么这个版本存在

v1.3.0 是 FCoP **协议语义层完整化**里程碑：

- **ADR-0031（GAL）**：让治理断层自发光——Solo Blindspot / CRITICAL 工具无 Review / 长期开放 Task 均会触发 ADMIN 告警
- **ADR-0032（fcop_audit）**：让协议合规缺口成为可执行文档——三场景体检 + 整改建议一体化

两者合并后，FCoP 三层语义执行链完整闭合：

```
ADR-0030 Schema Layer  →  能不能做（MCP Interceptor 阻断/放行）
ADR-0031 Signal Layer  →  发生了什么（治理告警自动升级）
ADR-0032 Compiler Layer →  该怎么修（体检报告 = 整改说明书）
```

## 新增内容

### fcop_audit（ADR-0032）— 协议状态编译器

**新增 1 个 MCP 工具**（总数 34 → 35）

#### `fcop_audit(scope, output, project_path)`

三场景协议体检工具。扫描项目，发现协议合规缺口，产出 INSPECTION 报告。

```
scope=new      — 新项目验收
scope=upgrade  — 版本升级后验收
scope=takeover — 老 non-fcop 项目首次引入（最完整扫描）
scope=auto     — 自动推断（推荐）
```

产出物：`fcop/shared/INSPECTION-{date}-{NNN}-{scope}.md`

报告格式（L3）：
- YAML frontmatter（机读）
- P0 阻塞 / P1 规范 / P2 整洁 违规分档
- **Execution Block**：按 Tier 1/2/3 分组的可复制整改命令建议

**关键语义**：INSPECTION = Structured Findings + Suggested Remediation Plan（不是执行指令）

#### Python API

```python
from fcop import Project, InspectionReport

proj = Project("./myproject")
report: InspectionReport = proj.audit(scope="auto", output="file")
print(report.overall_status)   # green / needs_remediation / blocked
print(report.p0_count)         # P0 阻塞性违规数
print(report.to_markdown())    # L3 完整报告
```

#### 六类扫描盲区（takeover 场景）

| 扫描方法 | 检测内容 |
|---|---|
| `scan_cursor_rules` | 协议规则文件缺失 + 草根 .mdc 混入 |
| `scan_shared_deployment` | 三层团队文档部署完整度 |
| `scan_misplaced_envelopes` | envelope kind: 与物理路径不符（Rule 2） |
| `scan_legacy_role_docs` | fcop/ 根目录草根角色书 |
| `scan_legacy_manifests` | fcop/ 非标准 JSON 双 manifest |
| `scan_ghost_prefixes` | DRAFT-/HANDOFF-/AMEND- 幽灵前缀文件 |

### GAL（ADR-0031）— 治理漂移检测与 ADMIN 告警

**新增 2 个 MCP 工具**（总数 32 → 34）

#### `fcop_list_alerts`

ADMIN 治理告警收件箱。读取 `fcop/alerts/ALERT-*.md`，支持 status / severity 过滤。

#### `fcop_create_alert`

手动归档治理缺口，写入新 ALERT-*.md 文件。

#### 三类漂移信号

| 信号 | 触发条件 | Severity |
|---|---|---|
| S1 `critical_tool_unreviewed` | 24h 内 CRITICAL_TAG 调用无 Review | high |
| S3 `missing_independent_verdict` | 执行窗口 > 6h 无治理事件（Solo Blindspot） | high |
| S4 `long_running_without_reconciliation` | open Task 超 24h 未归档 | low |

#### FCoP-Rule-G1（协议公理）

> `write_report / fcop_report` ∈ 执行域。自我叙述 ≠ 治理信号。  
> 只有 `write_review` / `mark_human_approved` / `fcop_check` 才构成独立治理视角。
