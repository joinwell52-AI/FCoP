# fcop + fcop-mcp 1.3.1 Release Notes

**发布日期**：2026-05-12  
**PyPI**：[fcop 1.3.1](https://pypi.org/project/fcop/1.3.1/) · [fcop-mcp 1.3.1](https://pypi.org/project/fcop-mcp/1.3.1/)

---

## 为什么这个版本存在

v1.3.0 发布后，**面向 Agent 的协议文档（规则文件 + 团队模板）集体停留在 v1.1**，导致 Agent 不知道 `fcop_audit()` / GAL / INSPECTION envelope 的存在，自然也不会使用它们。

v1.3.1 是**协议文档同步批次**，将所有 Agent 入口文档对齐到 v1.3.0 能力边界。

> 背景分析见 `.fcop/proposals/protocol-docs-sync-20260512.md`（P0 整改批次）。

---

## 变更内容

### fcop-rules.mdc → 2.3.0

**新增 Rule 9.6 · Protocol Inspection / 协议体检**（per ADR-0032）

- `fcop_audit()` 作为协议状态编译器的 Agent 行为规范
- INSPECTION 作为第 5 类 IPC envelope 的命名规则（`fcop/shared/INSPECTION-YYYYMMDD-NNN-{scope}.md`）
- 三场景使用规则（`new` / `upgrade` / `takeover` / `auto`）
- Execution Block 建议语义声明（不是协议指令）
- violation 分级规则：P0 blocking / P1 normative / P2 hygiene

**新增 Rule 9.7 · Governance Alert Layer / 治理告警层**（per ADR-0031）

- GAL 三类漂移信号定义（S1 / S3 / S4）
- FCoP-Rule-G1 协议公理：`write_report` ≠ 独立治理信号
- `fcop_list_alerts` / `fcop_create_alert` MCP 工具规范

### fcop-protocol.mdc → 2.1.0

- **Rule 9.6 Commentary**：三场景决策树 + INSPECTION 命名 + P0 处置原则 + Execution Block 语义声明
- **Rule 9.7 Commentary**：三类信号触发表 + FCoP-Rule-G1 实践含义 + 工具调用示例
- **批量整改授权模式**：proposal frontmatter `scope: batch-remediation` 机制（一次性批准 → 批量改 → 汇总 TASK+REPORT）
- `fcop/shared/` 目录表新增 `INSPECTION-` 审计产物行

### letter-to-admin.{zh,en}.md

摘要段从 "0.6.4 / 0.6.5" 全面重写为 **v1.3.0 摘要**，涵盖：
- `fcop_audit()` 接手老项目第一动作
- GAL 告警查看方式
- `risk_level` / `mark_human_approved` 风险管理
- 当前工具总数（35）和规则版本
