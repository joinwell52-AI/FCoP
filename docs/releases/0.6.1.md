# Release 0.6.1 — compat CLI for 0.5.x users

- **Date**: 2026-04-23
- **PyPI**: [`fcop 0.6.1`](https://pypi.org/project/fcop/0.6.1/) · [`fcop-mcp 0.6.1`](https://pypi.org/project/fcop-mcp/0.6.1/)

## What shipped

- **`fcop` wheel**: `[project.scripts] fcop = "fcop._compat_cli:main"`. Invoking `fcop` after upgrade prints migration guidance (use `fcop-mcp` / `uvx fcop-mcp`) and exits with status `1`. No library API change; ADR-0003 additive-only.
- **`fcop-mcp`**: version aligned in lockstep with `fcop` (no MCP tool contract change).

## Background

See [`docs/releases/0.6.0.md`](./0.6.0.md) §5.3 — `fcop` 0.6.0 had no console script, so upgraded 0.5.x users saw a bare `command not found`.

## Verify after upload

```bash
pip install -U fcop==0.6.1
fcop   # expect stderr message + exit 1
pip install -U fcop-mcp==0.6.1
fcop-mcp --help
```
